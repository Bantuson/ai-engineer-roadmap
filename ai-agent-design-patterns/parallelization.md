# Parallelization

- Split a large job into independent chunks that can be processed at the same time by multiple workers
- Includes retry logic for failed chunks and respects rate limits to avoid overwhelming systems

# When to use

- Web scraping
- Time-sensitive Ops
- Batch operations
- Web scraping
- Multi document analysis
- API aggregation
